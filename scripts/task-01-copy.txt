ADMIN DASHBOARD - TASK-01 TECHNICAL REQUIREMENTS
(Paste each line as a separate item in the Technical Requirements field)
================================================================

Build an ERC-20 token contract that blocks unverified users from minting or transferring tokens
Integrate with Redbelly's hasChainPermission(address) function from the Eligibility SDK
Users must complete KYC through the Individual Onboarding SDK before interacting with the token
Contract must revert with clear, descriptive error messages when unverified users attempt restricted actions
Configurable option allowing the contract owner to toggle whether transfers (not just minting) require verification
Admin function to update the eligibility contract address in case Redbelly deploys updated infrastructure
